Javascript诞生于1995年，当时它的主要目的是处理以前由服务器语言负责的一些输入验证操作。


javascript的组成

javascript由三部份组成：ECMAScript,DOM,BOM
ECMAScript规定了以下内容： 语法 类型 语句 关键字 保留字 操作符 对象

ECMAScript兼容

ECMAScript兼容由ECMA-262给出

1. 支持ECMA-262描述的所有“类型，值，对象，属性，函数以及程序句法与语义”。 2. 支持Unicode字符标准.

扩展

3. 添加ECMA-262没有描述的“更多类型，值，对象，属性和函数”。ECMA-262所说的这些新增特性，主要是指该标准中没有规定的新对象和对象的新属性。 4. 支持ECMA-262没有定义的“程序与正则表达式语法”。（也就是说，可以修改和扩展内置的正则表达式语法）

DOM

什么是DOM?

文档对象模型(DOM,Document Object Model)是针对XML但经过扩展用于HTML的应用程序编程接口(API, Application Programming Interface)

DOM做了什么？

DOM1的目标是把整个页面映射为了一个多层节点结构, 便于简化对文档中任意部分的访问和操作。DOM2的增加了很多模块：
DOM视图(DOM Views)：定义了跟踪不同文档(例如，应用css之前与之后的文档)视图的接口;
DOM事件(DOM Events)：定义了事件和事件处理的接口。
DOM样式(DOM Style)：定义了基于CSS为元素应用样式的接口。
DOM遍历与范围(DOM Traversal and Range)：定义了遍历和操作文档树的接口。
DOM3引入了统一方式加载和保存文档的方法——在DOM加载和保存(DOM Load and Save)模块中定义;新增了验证文档的方法——在DOM验证(DOM Validation)模块中定义。DOM3也对DOM核心进行扩展，开始支持XML1.0规范，涉及XML Infoset,XPath和XMLBase.

BOM

浏览器对象模型(BOM,Brower Object Model)可以对浏览器进行操作。功能如下: 弹出新浏览器窗口的功能。
移动，缩放和关闭浏览器窗口的功能。
提供浏览器详细信息的navigator对象。
提供浏览器所加载页面的详细信息的location对象。
提供用户显示分辨率详细信息的screen对象。
对cookies的支持。
像XMLHttpRequest和IE的ActiveXObject这样的自定义对象。

<script>

引用javascript的方法是在HTML中使用<script>标签引入。<script>定义了下列6个属性：
async: 可选，表示应该立即下载脚本，但不应妨碍页面中的其他操作。只对外部脚本文件有效。
charset: 可选，表示通过src属性指定的代码的字符集。（很少人使用）
defer: 可选，表示脚本可延迟到文档完全被解析和显示之后执行。
language: 已废弃。 src: 可选，表示包含要执行代码的外部文件。 type: 可选，表示编写代码使用的脚本语言的内容类型（也称为MIME类型）。

文档模式

有三种文档模式：混杂模式(quiks mode),标准模式(standards mode),准标准模式(almost standards mode)。不同文档模式会影响css渲染和js的解释执行
没有文档声明时，默认开启混杂模式。
标准模式：
